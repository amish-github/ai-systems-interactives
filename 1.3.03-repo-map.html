Reference Platform — Repository Map

Every module in the course uses this codebase. The five top-level folders map directly to the architecture diagram from Chapter 1.1. When you first clone the repository, open docs/ before anything else.

ai‑systems‑platform/ repository root
├── services/
One folder per running service — gateway, application service, and background workers. Each is a separate deployable unit with its own entry point.
│   ├── gateway/ │   ├── app‑service/ │   └── worker/
├── shared/
Code used by more than one service — data models, logging helpers, and the database client. Prevents the same logic from being copied into every service and drifting out of sync.
│   ├── models/ │   ├── logging/ │   └── db‑client/
├── infra/
Docker Compose files, container definitions, and configuration. Describes how the system is built and deployed so anyone can recreate the same environment from scratch.
│   ├── docker‑compose.yml │   └── config/
├── tests/
Automated tests for services and shared libraries. Your safety net — run the full suite before you change anything to establish a known-good baseline.
│   ├── services/ │   └── shared/
└── docs/
Top-level README, setup notes, and architecture references. The fastest way to orient yourself — read this folder before opening any service code.
    ├── README.md     └── architecture/
→

Map before you move. For each box on the Chapter 1.1 architecture diagram, identify the folder here that implements it and write down its one-sentence responsibility. When you can make that connection, finding any feature becomes a lookup, not a search.
