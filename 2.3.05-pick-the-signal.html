The three rules
Logs A specific single case — what exactly happened for one user, one request, one moment. High-cardinality detail (user id, request id) belongs here.
Metrics Aggregate trends and rates — how many, how often, how fast across all traffic. Labels must stay low-cardinality (status code, endpoint name — not user id).
Traces A cross-service path — where did one request go and where did it spend its time across services. Sample at high volume to keep storage costs manageable.
